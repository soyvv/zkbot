# zb-00012 OMS publish recorder JetStream events off the critical path

- Ticket: zb-00012
- Date: 2026-03-27
- Time: 15:47 +08
- Domain: oms, design
- Reporter: agent
- Status: resolved

## Description

OMS needs a dedicated recorder publish path for terminal-order and trade events on JetStream, but the publication must remain outside the single-writer mutation critical path.

## Symptoms

- Recorder design now depends on recorder-specific JetStream subjects for terminal orders and trades.
- Current OMS design and code publish low-latency order updates on plain NATS only.
- There is no explicit OMS ticket tracking the executor, subject contract, and payload-shape work needed for recorder-facing JetStream publication.

## Impact

Without this change, recorder cannot consume durable replayable OMS terminal-order and trade events as designed, or OMS may incorrectly add JetStream publish latency to the hot mutation path.

## Proposed Options Or Fixes

1. Add a recorder-specific publish executor in zk-oms-svc that queues JetStream publish work after writer mutation.
2. Define dedicated subjects for terminal-order and trade publication and document payload requirements for recorder.
3. Emit terminal-order events only on first transition into terminal state and emit per-fill trade events idempotently from OMS state transitions.
4. Keep plain NATS order_update fanout unchanged for strategies and monitors so low-latency consumers do not pay JetStream durability overhead.

## Open Questions

- Should recorder trade payload reuse the existing Trade proto directly or wrap it in a recorder-specific envelope with oms_id and source linkage fields? (yes, should enrich the data with related metadata/joining-data)
- Should JetStream use a single recorder stream with multiple subjects or separate streams for terminal orders and trades? (different streams for order and trade)
- What backpressure and failure policy should OMS apply when the recorder publish queue is full? (drop and log warning; later on will send out notification out)
