## Description

Recorder implementation is still missing for the new Postgres-first OMS recording path: consume JetStream terminal-order and trade events, persist end-state orders into a partitioned trd.order_oms table with 3-month retention, and run periodic partition rotation.

## Symptoms

- Recorder service design now assumes a Rust zk-recorder-svc consuming recorder-specific JetStream subjects.
- The base schema has trade_oms and snapshot tables, but no partitioned trd.order_oms table for terminal-order storage.
- There is no recorder runtime yet to upsert end-state OMS orders, persist fills from JetStream, or rotate monthly partitions.

## Impact

Without this work, the documented recorder design is not executable: terminal OMS order history is not persisted into the intended canonical table, retention is undefined, and post-trade services still lack the intended Postgres source of truth.

## Proposed Options Or Fixes

1. Add zk-recorder-svc as a Rust service with durable JetStream consumers for terminal-order and trade subjects.
2. Create trd.order_oms as a monthly range-partitioned table keyed by terminal timestamp with helper procedures for partition creation and old-partition drop.
3. Implement idempotent upsert logic for terminal orders and idempotent insert logic for trades in recorder-owned DB writers.
4. Add a periodic maintenance job in recorder to create current and next partitions ahead of time and drop partitions older than the 3-month retention window.

## Open Questions

- Should trd.order_oms store only terminal states or also accept late enrichment updates after terminal publication? (probably yes; log warning maybe)
- Do we want a dedicated order_event_oms append-only table in the first recorder milestone or defer that until after terminal-order persistence is stable? (no)
- Should partition maintenance live fully in SQL procedures, fully in recorder code, or use recorder code to invoke SQL helpers? (recorder should be mainly a scheduler; partition/rolling/TTL maintaining jobs, if possible, should be in pg)
