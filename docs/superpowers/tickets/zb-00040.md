## Description

While debugging Item 5 (OMS dispatch fix for IBKR), exploration found that cfg.instrument_refdata.instrument_id_exchange is written by the IBKR refdata loader (refdata.py:322) but is NOT read by the OMS, GW, or RTMD adapters. The OMS reads instrument_exch (mapped into Rust struct field instrument_id_exchange in db.rs:144 — confusing naming). The DB column instrument_id_exchange has no observed downstream consumer.

## Symptoms

- rg -n 'instrument_id_exchange' on Rust/Java/UI consumers shows no SELECT or join on the DB column itself (only the Rust struct field with the same name, which maps from instrument_exch).
- Loader-side: refdata.py:322 sets it to c.symbol (bare ticker) — duplicate of base_asset for STK rows.

## Impact

Wasted DB column + wasted writer cycles per refresh.||Confusing name overlap with the Rust struct field instrument_id_exchange (which is sourced from a DIFFERENT DB column, instrument_exch). Has caused tracing confusion already.

## Proposed Options Or Fixes

1. Audit all venue refdata loaders for who writes instrument_id_exchange. Confirm no consumer reads it. Drop the column via a Flyway migration; remove from venue loaders.
2. Or: rename the Rust struct field instrument_id_exchange -> instrument_exch_spec to remove the name collision, even if the DB column stays.

## Open Questions

- Are there any historical or experimental consumers (research notebooks, ad-hoc SQL) that read instrument_id_exchange? Run a 'grep across all repos + ops scripts' before dropping.

## Resolution

- Resolved At: 2026-04-29 00:15 +08
- Summary: Audit confirmed: cfg.instrument_refdata has NO instrument_id_exchange column (only instrument_exch). The Rust struct field of the same name is sourced from instrument_exch (db.rs:144) — purely internal naming, not a DB column. The dict key written by the IBKR loader at refdata.py:350 was silently dropped during SQL upsert, so it was truly dead weight. Removed both 'instrument_id_exchange' and 'asset_class' (also dead — codex review of Item 7 flagged it) from the IBKR loader output dict. Kept 'exchange_name' (read by zk-pyo3-rs/runner.rs:394 — Python strategy runtime bridge) and 'currency' (defensive). 186 IBKR tests green. No DB migration needed since no column existed. Rust struct field renaming (option b) deferred — current usage is contained and renaming would touch many Rust call sites.
