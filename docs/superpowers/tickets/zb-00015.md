# zb-00015 Pilot UI over-fetches live account and bot detail tabs

- Date: 2026-03-28
- Time: 21:55 +08
- Domain: pilot
- Reporter: Codex
- Status: resolved

## Description

Trading and bot management panels mount multiple live and historical queries at once, causing hidden-tab polling, retry amplification, and UI stalls under load.

## Symptoms

- Trading account detail mounts balances, positions, open orders, trades, history, and activities together with 2s polling
- Account detail page uses the same eager polling pattern even when only one tab is visible
- Fast-failing endpoints like trades or positions can multiply request volume because polling combines with React Query retry
- Backend positions path calls OMS with queryGw=true, so hidden-tab polling can drive live OMS/GW fan-out

## Impact

The UI can become unresponsive, backend request volume spikes, and Pilot/OMS/GW paths do unnecessary work for tabs the operator is not viewing. This creates operator-facing slowness and masks real backend performance issues behind client-side request amplification.

## Proposed Options Or Fixes

1. Gate live queries by active tab and selected row so only the visible live detail polls
2. Keep summary cards on a light cadence but make history and activities load-on-demand or manual refresh
3. Disable or sharply limit retry for fast-poll account and bot detail queries
4. Treat positions/open-orders/balances as separate refresh classes and avoid polling everything at the same cadence

## Open Questions

- Should bot live management adopt exactly the same active-tab refresh contract as trading accounts?
- Should the positions API expose a lighter cached path for UI polling instead of always forcing queryGw=true?

## Resolution

- Resolved At: 2026-03-28 21:58 +08
- Summary: Gated live polling by active tab across all three management panels. Live tabs (open-orders, trades, positions, balances) only poll when visible. Historical tabs (history, activities, executions) have no auto-refresh. Added retry:false for polling queries to prevent retry amplification. Files changed: useAccounts.ts, useBotDefinitions.ts, TradingPage.tsx, AccountDetailPage.tsx, BotsPage.tsx.
