# zb-00014 SchemaForm array fields lose user input — instruments sent as empty array

- Ticket: zb-00014
- Date: 2026-03-28
- Time: 15:55 +08
- Domain: pilot, engine
- Reporter: agent
- Status: open

## Description

SchemaForm renders array-type JSON Schema fields as JSON textareas. When the user enters values (e.g. instruments), the submitted config contains an empty array instead of the user's input. Multiple root causes identified and patched but the fix is not yet verified at runtime.

## Symptoms

- 1. User enters instrument codes in the engine config SchemaForm textarea during bot onboarding or editing.
2. The API request payload contains instruments: [] (empty array) despite user input.
3. The bot starts without any instrument subscriptions, making it non-functional.

## Impact

Bots created or edited via the UI may silently start with empty instruments/account_ids, leading to a non-functional engine that subscribes to nothing. Workaround: use the free-form JSON fallback (only available when engine schema is not registered).

## Proposed Options Or Fixes

1. Three patches applied (pending runtime verification):
1. buildDefaults: always JSON.stringify array values from defaults and schema defaults before storing as textarea initial value. Previously, actual JS arrays were passed to react-hook-form, which coerced them via .toString() into non-JSON strings.
2. hideSubmit value reporting: use getValues() inside the useEffect callback instead of the watch() snapshot captured at render time, ensuring the latest keystroke is always read from react-hook-form internal state.
3. processFormValues: handle all three cases for array fields — string (JSON.parse), actual array (pass through), anything else (default []).
If patches do not resolve, further investigation needed into react-hook-form watch()/register interaction with textareas and the useEffect firing order.

## Open Questions

- 1. Are all three patches necessary, or is one sufficient? Need runtime verification to isolate.
2. Does the same bug affect account_ids (also an array field)? Likely yes — same code path.
3. Should array fields use a tag-input UX instead of raw JSON textareas to avoid user confusion?
