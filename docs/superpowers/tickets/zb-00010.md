# zb-00010 Surface OKX derivative positions through gateway, OMS, and Pilot

- Ticket: zb-00010
- Date: 2026-03-26
- Time: 00:15 +08
- Domain: okx, positions, gateway, oms, pilot
- Reporter: agent
- Status: resolved

## Description

OKX derivative positions are not currently visible through the Pilot query path.

The OKX adapter already supports venue position query, but the live gateway/OMS
position query and publication flow is not fully wired through to Pilot in a way
that produces usable position visibility.

Need to complete the live position path for OKX and align it with the ongoing OMS
position/balance refactor direction.

## Symptoms

- OKX balances arrive and can be observed
- perp positions do not appear in Pilot query results
- position visibility is inconsistent with the expected exchange state

## Impact

- incomplete operator visibility for derivatives
- harder risk and exposure inspection during OKX onboarding
- balance-only visibility creates misleading account state

## Proposed Options Or Fixes

1. Implement real gateway `QueryPosition` using the OKX adapter.
2. Wire OMS startup/periodic position reconcile to ingest venue-backed OKX
   positions.
3. Ensure OMS query APIs expose the resulting position state to Pilot.
4. Add targeted tests for OKX position query and OMS position surfacing.
5. Keep the implementation aligned with the canonical position semantics tracked
   in the existing OMS position tickets.

## Acceptance Criteria

- OKX gateway returns venue-backed positions through `QueryPosition`
- OMS ingests and stores usable exchange-backed OKX position state
- Pilot query path shows OKX derivative positions consistently
